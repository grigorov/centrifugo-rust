# M3: JWT Connection Auth + Refresh

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or superpowers:executing-plans. Steps use `- [ ]`.

**Goal:** Authenticate WebSocket connections with JWT (HMAC/RSA/ECDSA), deriving the user id and info from claims, enforcing expiry, and supporting the REFRESH flow — verified against the real Go centrifugo v2.8.6.

**Authority (read directly):** `conformance/go-oracle/src/internal/jwtverify/token_verifier_jwt.go` and `internal/client/handler.go`. `docs/reference/protocol-v0.3.4-wire-format.md` for error/disconnect codes.

## Exact semantics (from the Go source)

`ConnectTokenClaims` = StandardClaims (`sub`, `exp`, `nbf`, `iat`, `iss`, `aud`, `jti`) plus:
- `info` (`json.RawMessage`, omitempty) — connection info.
- `b64info` (string, omitempty) — base64-StdEncoding of info; if present, **overrides** `info` (decoded bytes become Info).
- `channels` (`[]string`, omitempty) — server-side subscriptions (M4+; parse now, act later).

`VerifyConnectToken(tokenString)`:
1. Parse JWT. Select verifier by the **header `alg`**: HS256/384/512, RS256/384/512, ES256/384/512.
2. If no verifier configured for that alg → `errDisabledAlgorithm`; unknown alg → `errUnsupportedAlgorithm`. Both are signature-verification failures.
3. Verify signature. On failure → error.
4. Unmarshal claims. Check `exp` (IsValidExpiresAt) and `nbf` (IsValidNotBefore) vs now → if invalid → **`ErrTokenExpired`** ("token expired").
5. Build ConnectToken: `UserID = sub`, `Info = info` (or base64-decoded `b64info`), `Channels`, `ExpireAt = exp.Unix()`.

**Connect-flow mapping** (`handler.go` OnConnecting):
- `VerifyConnectToken` returns `ErrTokenExpired` → reply error **`ErrorTokenExpired` (109)**.
- Any other verify error → **`DisconnectInvalidToken` (3002)** close.
- Success → credentials with user = sub; if `ExpireAt > 0`, connection expires and the client must REFRESH.

**Insecure / anonymous:** `client_insecure` → no token required, user stays "" (current M1/M2 behavior). Without insecure and **no token** → centrifuge uses anonymous only if `client_anonymous` is set, else `DisconnectInvalidToken`/unauthorized. Confirm exact path in client.go connectCmd during impl.

**HMAC test key:** the Go tests sign with the literal key `secret` (HS256). Use the same in conformance.

### M3.4 — Connect flow auth
- `client.rs` `on_connect`: parse `ConnectRequest`; if `client_insecure` → current behavior (anonymous). Else:
  - empty token + not insecure → `DisconnectInvalidToken` (3002) (confirm vs client_anonymous path).
  - token present → `verifier.verify_connect_token`:
    - `Ok(ct)` → `self.user = ct.user`; store `expire_at`; ClientInfo conn_info from `ct.info`; reply ConnectResult (with `expires`/`ttl` if expire_at>0).
    - `Err(Expired)` → reply `Error::token_expired()` (109).
    - `Err(Invalid)` → `CommandOutcome::disconnect(Disconnect::invalid_token())` (3002).
  - The publisher ClientInfo on PUBLISH now carries the authed user + conn_info.
- Commit.

### M3.5 — REFRESH flow
- `client.rs` `on_refresh` (replace the `not_available` stub): verify the new token; `Ok` → update expire_at, reply `RefreshResult { client, version, expires, ttl }`; `Expired`/`Invalid` per handler.go (expired during refresh → ErrorExpired/Disconnect; confirm). SUB_REFRESH stays stub until M6 private channels.

## Risks
- **jsonwebtoken default validation** differs from Go's manual exp/nbf check (leeway, required claims). Configure `Validation` explicitly (no required `aud`, disable leeway) and map error kinds precisely so expired→109 and bad-sig→3002 exactly match Go.
- **Disabled vs unsupported alg** both surface as invalid-token close (3002) in the connect flow — fine, but keep the distinction internally.
- **b64info overrides info** — easy to miss; covered by a test.
- **Connection expiry / TTL fields** in ConnectResult must match Go when `exp` is set (expires=true, ttl=secondsUntilExp). Verify via golden with an exp-bearing token.
